串口设置布局重构方案

当前问题：缩小窗口后串口配置行显示不全，6个下拉框+2个按钮挤在一行

当前布局（问题展示）

窗口宽度 ~800px 时
Port: Baud: Data: Par: Stop: Flow: ● Disconnected

⚠ 12个控件挤在一行，缩小到~900px就开始裁剪/溢出

三种重构方案

A

方案A：串口设置移入"总览"页签顶部

将 ConnectionPanel 从全局顶部栏移至总览页签内最上方，简化顶部只保留连接状态指示条

总览 TAB（窗口缩小时）
串口配置：
仪表盘 / 实时数据区域
日志面板

优点

  • 不需新增页签
  • 串口配置靠近数据展示区，逻辑聚合
  • 其他功能页签界面更干净
  • 改动量最小

缺点

  • 配置与数据混合在同一页面
  • 连接后需隐藏配置区避免占空间
B

方案B：串口设置独立为"连接设置"页签并置顶

新增一个 TAB 页叫"连接设置"，放在所有功能页签之前（默认显示），原来的串口配置行改为简洁状态条

TabBar 排列
🔌 连接设置 总览 电流控制 温度控制 ...

在"连接设置"页签中提供宽敞的串口配置区域

串口：
波特率：
数据位：
校验位：
停止位：
流控制：

优点

  • 各功能页签完全不受串口配置干扰
  • 配置页签空间充足，可用FormLayout规范排列
  • 视觉层次清晰，分隔明确

缺点

  • 多一个页签（总计8个），TabBar可能拥挤
  • 连接状态在另一个页面，操作时需切换回来看
C

方案C：顶部仅保留"连接状态条"，串口设置折叠在侧边栏抽屉中

顶部只显示一行简洁状态条（已连接/已断开 + IP/端口信息），点击展开侧边抽屉面板进行串口配置

紧凑模式
🔌 COM3 @ 115200 ● 已连接
展开后的配置面板（抽屉/弹出）
串口：COM3 ▾
波特率：115200 ▾
数据位：8 ▾
校验位：None ▾
停止位：1 ▾
流控制：None ▾
Tab 功能区域

优点

  • 极致紧凑，窗口最小化时体验最好
  • 状态信息始终可见
  • 现代感的抽屉交互

缺点

  • 实现复杂度最高（需抽屉动画/QPropertyAnimation）
  • 配置操作多了一步点击
  • 不符合用户现有操作习惯
